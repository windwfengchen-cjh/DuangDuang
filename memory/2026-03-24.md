## Fixed Requirement/Problem Recognition Logic (2026-03-24)

### Problem
Previous logic did not properly distinguish between "跟进这个需求" and "跟进这个问题", causing confusion between requirement follow-up and problem tracking workflows.

### Solution
Updated recognition logic in `MEMORY.md` with clear keyword-based rules:

| Keyword | Trigger | Target Table |
|---------|---------|--------------|
| 需求/新功能/项目/功能 | requirement-follow | 需求跟进清单 (Op8W...) |
| 问题/bug/故障/异常/报错 | 问题跟进流程 | 业务反馈问题记录表 (KNiib...) |

### Recognition Flow
```
User quotes message + says "跟进..."
  ↓
Check for keywords:
  - "需求" → requirement-follow workflow
  - "问题" → problem tracking workflow
  - Neither → Ask for clarification
```
