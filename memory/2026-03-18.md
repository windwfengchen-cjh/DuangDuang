## 技术要点记录

### Resource API vs Images API
- **Resource API**: `/im/v1/messages/{message_id}/resources/{file_key}?type=image`
  - 优点: 可下载任何人发的图片
  - 需要: message_id + file_key
- **Images API**: `/im/v1/images/{image_key}`
  - 限制: 只能下载自己发的图片
  - 需要: image_key

### 图片转发流程
1. 接收带图片的消息
2. 提取 message_id 和 image_key
3. 使用 Resource API 下载原图
4. 上传到飞书获取新的 image_key
5. 发送到目标群
6. 自动清理本地临时文件
