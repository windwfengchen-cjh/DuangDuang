# requirement-followup Skill

## 功能
自动化需求跟进流程，包括：
- 从飞书表格查询待处理需求
- 获取需求人联系方式
- 私聊/建群多渠道沟通
- 多轮信息收集（7个关键问题）
- PRD文档自动生成
- 表格状态自动更新

## 适用场景
收到"跟进XX需求"指令时使用

## 核心特性

### 1. 智能沟通模式切换
```
[开始]
  ↓
[查询表格] → 获取需求信息
  ↓
[尝试私聊需求方]
  ↓
私聊成功？
  ↓是          ↓否
[私聊收集]   [创建群聊/通知Boss创建]
  ↓              ↓
[群内收集] ← [Boss拉人进群]
  ↓
[收集7个问题]
  ↓
[确认信息完整？]
  ↓是          ↓否
[写PRD文档]   [继续追问/补充]
  ↓
[更新表格状态]
  ↓
[向Boss汇报]
  ↓
[结束]
```

### 2. 群聊沟通方案
当飞书机器人无法直接私聊用户时（权限限制），系统会：
1. 自动创建临时需求沟通群
2. 邀请Boss加入
3. 提示Boss拉需求人进群
4. 在群内收集需求信息

### 群聊消息监听规则

在需求调研群中，需要**主动监听**群消息，而不仅是被@才响应：

**规则**：
- 即使需求人没有@机器人，也要关注其发言内容
- 当需求人回答调研问题时，主动记录和整理
- 当需求人提出新问题时，及时响应

**实现方式**：
- 通过飞书事件订阅监听群消息
- 识别需求人发言（根据飞书ID匹配）
- 自动提取关键信息并记录

### 3. 七问需求收集法
使用 `prd-document` skill 的标准7个问题：
1. **业务背景** - 需求产生的场景和原因
2. **目标用户** - 需求的使用对象
3. **现状描述** - 当前系统/流程现状
4. **核心痛点** - 当前遇到的问题
5. **期望解决方案** - 希望实现的功能
6. **优先级和时间** - 期望的完成时间
7. **相关资料** - 数据、截图、文档等

## 执行流程

### 整体流程

```
用户指令跟进需求
    ↓
查询飞书表格获取需求详情
    ↓
获取需求人联系方式
    ↓
尝试私聊（可能被限制）
    ↓
如果私聊失败:
    创建临时需求沟通群
    邀请Boss加入
    提示Boss拉需求人进群
    ↓
Boss拉人进群后
    ↓
在群内收集需求信息（多轮对话）
    ↓
确认需求细节完整？
    ↓ 是                      ↓ 否
使用prd-document skill    继续追问/补充信息
生成PRD文档                    ↓
    ↓                    [直到信息完整]
更新表格状态
    ↓
向Boss汇报结果
```

### 群聊模式流程

```
[Boss拉需求人进群]
  ↓
[开始监听群消息]
  ↓
[需求人发言（无论是否@）]
  ↓
[识别发言内容]
  ↓
是调研回答？ → [记录信息]
是提问？ → [及时响应]
是闲聊？ → [忽略]
```

**关键原则：先收集，后写文档**
- ❌ 错误：一开始就写文档框架，再收集信息填充
- ✅ 正确：先收集完整信息，确认后再写文档

## 重要规则

### 规则1：先收集，后写文档
**禁止在需求细节未确认前写PRD文档。**

正确顺序：
1. 联系需求方（私聊/群聊）
2. 收集完整需求信息（7个问题）
3. **确认需求细节完整**
4. 使用 prd-document skill 生成PRD文档

### 规则2：群聊创建失败时的处理
如果API无法直接创建群聊：
1. 通过私聊通知Boss需要创建群聊
2. 请Boss手动创建并拉需求方进群
3. 等待进群后再开始收集

### 规则3：需求确认清单
写文档前必须确认以下信息已收集：
- [ ] 业务背景清晰
- [ ] 目标用户明确
- [ ] 现状描述完整
- [ ] 核心痛点具体
- [ ] 期望方案可行
- [ ] 优先级和时间确定
- [ ] 相关资料（接口文档、产品清单等）已提供

## 使用方式

### 方式1：命令行调用

```bash
# 列出待处理需求
python requirement_followup.py --list

# 跟进指定需求（自动选择沟通模式）
python requirement_followup.py --id rec_001

# 指定Boss ID，私聊失败时自动创建群聊
python requirement_followup.py --id rec_001 --boss <boss_id>

# 根据标题关键词跟进
python requirement_followup.py --title 导出

# 批量处理所有需求
python requirement_followup.py --batch

# 创建需求沟通群
python requirement_followup.py --group "需求名称" <boss_id>

# 在群内收集信息
python requirement_followup.py --collect <chat_id> <需求人ID>
```

### 方式2：Boss指令触发

Boss可以通过以下方式触发需求跟进：

```
跟进需求                    # 列出所有待处理需求
跟进需求 rec_001           # 跟进指定记录ID的需求
跟进需求 导出              # 跟进标题包含"导出"的需求
```

## 飞书私聊限制说明

### 限制原因
飞书机器人存在以下权限限制：
- ✅ 可以创建群聊
- ❌ 不能直接给未授权用户发送私聊消息
- ✅ 可以生成群二维码/链接

### 解决方案
当检测到私聊受限时，系统会自动切换到群聊模式：
1. 创建临时需求沟通群
2. 生成群分享链接
3. 通知Boss拉人进群
4. 群内进行需求收集

### 配置要求
使用群聊模式需要提供Boss的飞书ID：
```bash
python requirement_followup.py --id rec_001 --boss ou_xxxxxxxx
```

## 常见错误

### 错误1：未收集信息就写文档
❌ 错误：先创建PRD框架，再收集信息填充  
✅ 正确：收集完整信息后，一次性生成PRD

### 错误2：群聊创建失败就停止
❌ 错误：API创建群聊失败就放弃  
✅ 正确：通知Boss手动创建，继续跟进

### 错误3：信息不完整就写文档
❌ 错误：部分信息缺失就开始写PRD  
✅ 正确：使用需求确认清单，确保完整后再写

## 故障排查

### 问题：找不到联系人
**解决**：确保 feishu_contacts.json 文件存在且包含反馈人信息。

### 问题：无法查询表格
**解决**：检查 .feishu_bitable_config.json 中的配置是否正确。

### 问题：私聊发送失败
**解决**：这是预期行为，系统会自动切换到群聊模式。需要提供Boss ID。

### 问题：无法创建群聊
**解决**：检查飞书API权限，确保机器人有创建群聊的权限。

### 问题：文档生成失败
**解决**：检查 docs/prd/ 目录是否有写权限。
