## 二、关键规则

### 规则1：先收集，后写文档（铁律）

**❌ 禁止**：需求细节未确认前就写PRD文档  
**✅ 正确**：收集完整信息 → 确认完整 → 再写文档

### 规则2：群聊消息监听

在需求调研群中，**即使需求人没有@机器人**，也要：
- 关注其发言内容
- 识别调研回答并记录
- 及时响应提问

## 三、群聊沟通详细流程

### 场景：私聊受限（错误码400）

```
私聊失败(400错误)
      ↓
创建群聊（包含Boss）
      ↓
在群里@Boss，提示拉需求方进群
      ↓
Boss拉需求方进群
      ↓
开始监听群消息
      ↓
需求方发言（无论是否@）
      ↓
识别内容 → 记录/响应
      ↓
信息收集完整
      ↓
生成PRD文档
```

### 群聊消息处理逻辑

```python
if 发言人是需求方:
    if 内容是调研回答:
        记录到需求信息库
    elif 内容是提问:
        及时响应解答
    elif 内容是闲聊:
        忽略
else:
    忽略（或视情况响应）
```

## 四、需求确认清单

写PRD文档前必须确认以下信息已收集：

- [ ] **业务背景清晰** - 知道为什么要做这个需求
- [ ] **目标用户明确** - 知道为谁做
- [ ] **现状描述完整** - 了解当前情况
- [ ] **核心痛点具体** - 明确要解决的问题
- [ ] **期望方案可行** - 知道要达到什么效果
- [ ] **优先级和时间确定** - 明确紧急程度和上线时间
- [ ] **相关资料齐全** - 接口文档、产品清单、认证方式等

**只有以上全部确认，才能开始写PRD文档！**

## 五、多轮对话详解

### 什么是一轮对话？

**一轮对话** = 提问 → 回答 → 追问澄清 → 确认理解

### 多轮对话示例

**第一轮：了解背景**
```
AI：请先介绍一下这个需求的业务背景？
需求方：我们要接入杭州臻选的运营商下单接口。
AI：是为了解决什么问题呢？是新增合作伙伴还是替换现有渠道？
需求方：是新增合作伙伴，杭州臻选是我们的新合作方。
AI：明白了，是为了拓展新合作渠道。那主要服务的用户群体是？
...（继续追问直到背景清晰）
```

**第二轮：分析问题**
```
AI：了解背景后，想了解一下当前系统的现状？
需求方：目前我们还没有和杭州臻选对接过。
AI：那现有的运营商对接流程是怎样的？可以简单描述一下吗？
...（追问现状细节）
```

**第三轮：探讨方案**
```
AI：基于前面的了解，您期望的解决方案是怎样的？
需求方：希望能自动把订单推送到杭州臻选系统。
AI：具体需要推送哪些信息？订单状态如何同步回来？
...（深入探讨方案细节）
```

### 每轮对话的结束标准

一轮对话可以结束的标志：
- ✅ 该主题的信息已收集完整
- ✅ 没有明显遗漏或疑问
- ✅ 需求方确认"就这些了"

### 对话技巧

1. **先倾听，后提问** - 让需求方充分表达
2. **复述确认** - "您的意思是...对吗？"
3. **追问细节** - "具体是指...？"、"能否举个例子？"
4. **总结记录** - 每轮结束简要总结已确认的信息
5. **渐进深入** - 从宏观到微观，逐步细化

## 六、常见问题处理

### Q1: 私聊失败怎么办？
**A**: 创建群聊，请Boss拉需求方进群。

### Q2: 需求方迟迟不回复怎么办？
**A**: 
1. 在群里友好提醒
2. 如果超过24小时无响应，汇报Boss
3. 由Boss决定是否电话/线下跟进

### Q3: 信息收集不完整怎么办？
**A**: 
1. 继续追问缺失的信息
2. 明确告知需要哪些信息才能继续
3. 不要勉强写不完整的PRD

### Q4: 需求方中途变更需求怎么办？
**A**:
1. 记录变更内容
2. 确认变更后的完整需求
3. 重新收集信息（如有必要）
4. 更新PRD文档

### Q5: 一轮对话问完7个问题可以吗？
**A**: ❌ 不可以！
- 7个问题应该分多轮对话逐步深入
- 每轮根据上一轮回答追问细节
- 确保每个问题都理解准确、记录完整
- 通常需要3-5轮对话才能收集完整
